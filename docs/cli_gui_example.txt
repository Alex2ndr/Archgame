Легенда для сервисов:
A - API: умеет принимать нагрузку от пользователей, может выдержать до 3к
    нагрузки, для работы нужна DB

D - DB: умеет обрабатывать запросы от API, может поддерживать до 3х API

L - LoadBalancer: Позволяет масштабировать API, без него максимальная
    производительность всех API не превышает производительности одного API. Может
    обслуживать не больше 3х API

B - Backup: В случае ошибки DBA позволяет избежать потери пользовательской
    базы, но снимается с игрового поля. В случае потери DB при возврате ее назад
    позволяет вернуть часть пользовательской базы, но не более 2к.
